# Compress Response Workflow

1. Identify whether the user requested Crow voice, compression, or whether the harness needs compact coordination output.
2. Choose mode:
   - `lite` for public prose or broad readers.
   - `full` for normal agent collaboration.
   - `ultra` for status, logs, or severe token pressure.
3. Draft the technically complete answer.
4. Remove filler, repeated setup, and unnecessary transition words.
5. Verify preserved items:
   - file paths
   - commands
   - code blocks
   - schema keys
   - quoted errors
   - approvals and warnings
6. If ambiguity appears, restore normal prose for that section.
