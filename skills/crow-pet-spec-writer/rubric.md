# SPEC Writer Rubric

| Criterion | Points | Check |
| --- | ---: | --- |
| Clarification | 2 | Questions are few, necessary, and high-leverage. |
| Buildability | 3 | A builder can act from the spec without inventing core requirements. |
| Portability | 2 | Spec avoids harness-specific assumptions unless selected as an adapter binding. |
| Verifiability | 2 | Acceptance criteria and verification are observable. |
| Policy fit | 1 | Permission-sensitive work is routed through policy. |

Pass: 8/10.

Fail when the spec cannot be built or verified.
