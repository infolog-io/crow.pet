# CI/CD Playbook

Reusable lessons for checks, logs, and deployment-related tasks.

## STRATEGIES & INSIGHTS

[read-first-0001] helpful=0 harmful=0 :: Inspect failing checks and logs before proposing code or configuration changes.

[approval-0001] helpful=0 harmful=0 :: Treat pushes, pull requests, rollbacks, promotions, and environment changes as approval-gated external actions.

## COMMON MISTAKES TO AVOID

[release-bias-0001] helpful=0 harmful=0 :: Do not route general artifact or adapter work through CI/CD language unless the task is actually about checks or deployment.
